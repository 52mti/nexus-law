# Nexus Law (汇动法律 AI) - Global Architecture & Prompt Guide

## Project Overview
**Nexus Law** is an AI-powered legal document and case management platform built with modern React ecosystem. It provides legal researchers, lawyers, and compliance officers with intelligent tools for document generation, case search, legal consultation, and compliance review.

**Primary Language**: Chinese (Mandarin)  
**Current Status**: Active Development  
**Target Users**: Legal professionals, document analysts, compliance teams

---

## 1. Technology Stack & Dependencies

### Core Framework
- **React**: v19 (Hooks-based functional components)
- **TypeScript**: Strict mode enabled for type safety
- **Vite**: Modern bundler with fast HMR
- **React Router DOM**: v7 for client-side routing

### Styling & UI Components
- **Tailwind CSS**: v4 (primary), utility-first styling for layout, spacing, responsive design
- **Ant Design (antd)**: v6 (secondary), for complex components (Table, Menu, Layout, Modal, Form, Pagination)
- **Custom Icons**: `@ant-design/icons` throughout the app
- **Animation**: CSS animations with Tailwind (`animate-fade-in`)

### Internationalization & Localization
- **i18next**: v25 for multi-language support
- **react-i18next**: v16 for React integration
- **Language Detector**: Browser auto-detection enabled
- **Current Locale**: Chinese (zh-CN) by default, English (en-US) as fallback

### Additional Libraries
- **axios**: HTTP client for API calls
- **qrcode.react**: QR code generation for payment flows
- **react-markdown**: Markdown rendering for legal documents
- **remark-gfm**: GitHub-flavored markdown support

## 4. Routing Architecture

### Route Structure

```
- /login             → AuthPage (independent, no MainLayout)
- /                  → MainLayout wrapper
  ├── /chat          → AIChatPage
  ├── /history       → HistoryPage
  ├── /doc           → DocPage
  ├── /legal-search  → LegalSearchPage
  ├── /case-search   → CaseSearchPage
  ├── /case-review   → CaseReviewPage
  ├── /compliance    → CompliancePage
  ├── /account       → AccountInfoPage
  ├── /membership    → MembershipPage
  ├── /orders        → OrderListPage
  └── /points        → PointsRecordPage
```

### Configuration Details
- **Auth Routes** (`/login`): Rendered standalone outside MainLayout
- **Protected Routes**: All other routes wrapped in `MainLayout` with Sidebar, Header, and Portal slot
- **Secondary Sidebar Pattern**: Pages needing sub-navigation (e.g., `DocPage`) inject content via `<PortalSidebar>` into the `#secondary-sidebar-portal` slot in MainLayout
- **Default Redirect**: Root `/` redirects to `/chat`

## 6. Page Container Pattern

### PageContainer Component
Located: `src/components/layout/PageContainer.tsx`

**Purpose**: Standardized wrapper for all data/list pages to ensure consistent styling, spacing, and layout constraints.

**Props**:
```typescript
interface PageContainerProps {
  children: React.ReactNode;
  maxWidth?: 'max-w-5xl' | 'max-w-6xl' | 'max-w-7xl' | 'max-w-[1200px]' | 'max-w-[1440px]' | 'none';
  className?: string;
}
```

**Default Styling**:
- Background: `#f9fafb` (light gray)
- Padding: `p-6` (1.5rem)
- Height: `min-h-full` (fills vertical space)
- Flex: `flex flex-col` (column layout)
- Animation: `animate-fade-in` (fade in on mount)
- Max-width: `1200px` (golden reading width) by default, centered with `mx-auto`

**Usage**:
```tsx
import { PageContainer } from '@/components/layout/PageContainer';

export const MyPage = () => {
  return (
    <PageContainer maxWidth="max-w-[1440px]">
      {/* Page content here */}
    </PageContainer>
  );
};
```

**Pages Using PageContainer** (as of latest):
- HistoryPage
- OrderListPage
- PointsRecordPage

## 8. Styling Guidelines

### Primary Method: Tailwind CSS
- Use Tailwind for: Layout, spacing, responsive design, hover effects, borders, shadows
- Naming convention: Utility-first (`flex`, `justify-center`, `gap-2`, etc.)
- Responsive breakpoints: `sm:`, `md:`, `lg:`, `xl:`, `2xl:`
- Custom spacing: Use standard scale (p-1 = 0.25rem, p-6 = 1.5rem, etc.)

### Secondary Method: Ant Design Components
- Use Ant Design for: Complex components (Table, Menu, Modal, Form), Form validation, Data management
- Styling Ant Design: Override with Tailwind classes or CSS-in-JS via `className` prop
- Theme Configuration: Centralized in `App.tsx` ConfigProvider token config

### Common Styling Patterns

**Card/Container Style**:
```tsx
className="bg-white rounded-2xl shadow-sm border border-gray-100 p-6"
```

**List Item Hover**:
```tsx
className="cursor-pointer hover:bg-[#f0f2f7] transition-all rounded-lg"
```

**Text Hierarchy**:
```tsx
// Primary heading
className="text-[16px] font-bold text-gray-800"

// Secondary text
className="text-[14px] text-gray-600"

// Muted text
className="text-[12px] text-gray-400"
```

**Button Style**:
```tsx
className="px-4 py-2 rounded-lg bg-[#5c6bc0] text-white font-medium hover:bg-[#585ee6] transition-colors"
```

**Status Indicators**:
```tsx
// Dot indicator
className="w-2 h-2 rounded-full bg-[#10b981]" // success

// Text with status
className="flex items-center gap-2 text-emerald-600 font-medium"
```

### Responsive Design Approach
- **Mobile-first**: Start with base utility, layer with responsive prefixes
- **Common breakpoints used**:
  - `md:` (768px) - Tablet/small laptops
  - `lg:` (1024px) - Desktops
  - `xl:` (1280px) - Large screens
  - Custom: `[width]` for specific values

Example:
```tsx
className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4"
```

## 10. TypeScript Standards

### Type Definitions
- **Interface vs Type**: Prefer `interface` for object shapes, `type` for unions/aliases
- **Prop types**: Define as interfaces in same file or `types/` directory
- **Strict mode**: Enabled in `tsconfig.json`, enforce non-null assertions sparingly

### Common Type Patterns

```typescript
// Component Props
interface PageComponentProps {
  title?: string;
  onAction?: (id: string) => void;

// Data Models
interface OrderRecord {
  key: string;
  orderId: string;
  status: 'pending' | 'success' | 'cancelled';
  amount: number;
  orderTime: string;
}

// API Response
interface ApiResponse<T> {
  code: number;
  message: string;
  data: T;
}
```

## 11. Common Implementation Patterns

### Modal/Dialog Pattern
```typescript
const [isOpen, setIsOpen] = useState(false);
const [selectedItem, setSelectedItem] = useState<Item | null>(null);

const handleOpen = (item: Item) => {
  setSelectedItem(item);
  setIsOpen(true);
};

return (
  <>
    <MyModal open={isOpen} onCancel={() => setIsOpen(false)} item={selectedItem} />
  </>
);
```

### Table with Actions Pattern
```typescript
const columns = [
  { title: '名称', dataIndex: 'name', key: 'name' },
  {
    title: '操作',
    key: 'action',
    render: (_, record) => (
      <span 
        className='text-[#666cff] cursor-pointer hover:text-[#585ee6]'
        onClick={() => handleAction(record)}
      >
        操作
      </span>
    ),
  },
];

<Table columns={columns} dataSource={data} />
```

### Timeline/History Pattern
```typescript
mockData.map((group, index) => (
  <div key={group.id} className="flex relative pb-10">
    {/* Timeline line (skip for last item) */}
    {index !== mockData.length - 1 && (

    )}
    
    {/* Timeline dot and content */}
    <div className="w-24 shrink-0 flex items-start justify-between relative z-10">
      <span className="text-gray-500">{group.date}</span>
      <div className="w-[7px] h-[7px] rounded-full bg-[#666cff] ring-[5px] ring-[#f9fafb]" />
    </div>
    
    {/* Grid content */}
    <div className="flex-1 grid grid-cols-1 md:grid-cols-2 xl:grid-cols-3 gap-5">
      {/* Cards */}
    </div>
  </div>
))
```

## 21. Important Notes & Gotchas

### Known Patterns
1. **Secondary Sidebar**: Uses React Portals to avoid layout shifts. Always check `#secondary-sidebar-portal` div exists in MainLayout.
2. **PageContainer**: Wraps page content, already includes bg color and padding. Don't duplicate these on inner wrappers.
3. **Theme Color**: `#5c6bc0` is the primary brand color, used for buttons, links, active states. Alternative hover: `#585ee6`.
4. **Custom Tailwind Colors**: Configured in `tailwind.config.ts` or used inline with bracket notation (e.g., `bg-[#5c6bc0]`).

### Anti-Patterns to Avoid
- ❌ Don't wrap entire page in another `div` with background/padding (use PageContainer instead)
- ❌ Don't hardcode color hex values; use Tailwind classes or defined CSS variables
- ❌ Don't add className overrides to Ant Design components without understanding cascading
- ❌ Don't skip TypeScript type definitions for props and state
